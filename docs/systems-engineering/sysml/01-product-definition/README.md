# 01 Product Definition

This level captures why the product exists, what users must accomplish, and what product requirements follow from that intent.

- [product-definition.sysml](./product-definition.sysml) defines the product mission, source-of-truth rule, workbench promise, traceability scope, multi-context scope, and current strategic choice.
- [use-cases.sysml](./use-cases.sysml) groups the product-purpose use cases that drive requirements and decomposition.
- [requirements.sysml](./requirements.sysml) defines product requirements and links them to use cases, functions, and product blocks.

Use this folder when refining product purpose, user goals, requirements, or scope boundaries.
